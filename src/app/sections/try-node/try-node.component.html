Try Node:

At last, we have the TryNode. It receives two arbitrary ASTNodes, one for its body, and the other one for the error-state.
In a perfect world there would be no need for a TryNode, but as previously mentioned, the execution of individual tasks could also possible end up in an error-state. One reason for that could be because the bot couldn't get to a specified target because it floats up in the air, or because the bot tried to take items from a chest which was empty. Normally the bot would just be stuck now, because the actual task couldn't be finished. But with the introduction of the TryNode, which adds new error-transitions to all the actions in its body-note, which could possibly fail.
Should an action throw an error, the program continues on with a predefined error-node, instead of being stuck in the failed state. This allows the programmer to catch possible errors, and tell the bot what it should do, if the actual task couldn't be completed.
