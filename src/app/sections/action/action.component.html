What is an Action?

An Action is the foundation block of our compiler. In other words it is the actual class that tells the minecraft bot what to do. As you can see from the UML Diagram bellow, it extends the State behaviour interface of the mineflayer-api and additionally forces the onStateEntered() and onStateExited() methods on all its children.

When the minecraft-bot is currently transitioning into a new state, the onStateEntered() function of the action class is called. This function can be asynchronous, and it tells the bot what it should do, and awaits the result of this task.
When the task could be completed, the function tells the state-machine that the current task is finished, and that the next transition can be executed.
If the action failed, the errored-Flag will be set.
When the bot leaves the current state, the onStateExited() function is called, which primarily resets all internal states of the corrisponding action class.

As you can see from the code, the Action classes are very minimalistic, and don't contain much boilerplate code.
If someone wants to extend the minecraft-bot with new features, he can just add new action-classes and new nodes and implement the onStateEntered() and onStateExited() functions, thats it. Everything else will work out just fine.
