What is a State-Machine?

A state-machine is an abstract machine, that can be in exactly one state at a given time.
This statement is very important because that means the follwing state can be uniquley determined using the current state and the new input.
Such a state machine is also called a deterministic finite automaton, and the fact that it uniquely defines the current state and any possible future state alows us to build a minecraft-bot with it.
By defining all the possible states and transitions between those states we can 'hardcode' our program, so that that everything, that could possible happen in our program has a unique state and transitions to get, and exit from there.

In contrast to a determinsitc finite automaton (DFA) exists the nondeterminstic finite automaton (NFA) which is way more compicated. Such an automaton can be in multiple states at a given time and use multiple transitions at once or use ϵ-transitions to instantely transition to a new state, even if there is no input.

Such automata aren't very useful for minecraft bots, because theire nature of being in multiple states at a given time isn`t very applicable for a minecraft bot, and would need a very complicated system to ultimate determit what the bot should do.

Here are some videos of a DFA that accepts the input (shown by landing in the accepting Z0-State) if the given binary-input-string is a multiple of 3.
The autmata takes the input string, for example: "01011" and processes it letter by letter.
It then takes the according transition, based on the value of the current character and goes into the new state.
If no more letters are available and the current state ended up in the accepting-state Z0, the automata accepts the input, and ensures that the input number was a multiple of 3.
