While Node:

Next we have the WhileNode. It receives a ConditionNode and a body-node, which can be every possible ASTNode.
Its job is to repeat the given body-node until the condition of the ConditionNode is no longer true.
When you compile a WhileNode, it compiles its body-node and hooks it up with its internal states.
Specifically it creates 3 internal transitions which serve the following purpuses:

  1. The "enter-loop" transition connects the starting-state of the WhileNode to the entry-state of the body. This transition occurs when the condition of the ConditionNode evaluates to true. When this transition occures the program flow is directed to the body-node, where the program continous.

  2. Once the body note is finished executing. The "exit-loop" transition occurs. This transition allows the program flow to reenter into the head of the WhileNode once the body is finished.
    There the loop-condition gets evaluated again, and the program flow enters the body node again or exits the WhileNode entirely, depending on the result of the evaluation.

  3. The last transition is the "exit-while-node" transition. It occurs when the condition of the ConditionNode evaluates to false. That means that the while-loop is finished and the program flow can continue with the next node.
