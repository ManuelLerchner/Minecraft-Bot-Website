Task Node:

The Task Node is the simplest node. It's modeled as an abstract class as you can see in the UML-Diagram. Its direct children then implement the actual logic.
When you compile it, it just returns the action associated with it, and whether that action can possibly fail, that's it. At this level there are no transitions yet because we're dealing with a single actions, in other words we are handling a single states of the state-machine.

An example of a TaskNode is for example a GoToNode which tells the bot to go to the specified coordinates.
